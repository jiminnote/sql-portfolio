# Explanation: 잡은 물고기 중 가장 큰 물고기의 길이 구하기

## 1. 접근 아이디어

이 문제는 `FISH_INFO` 테이블에 저장된 물고기 길이 중 **가장 큰 값(MAX)** 을 조회한 뒤,  
결과를 `"XXcm"` 형태의 문자열로 변환하는 매우 단순 명확한 유형이다.

SQL의 `MAX()` 집계 함수는 다음과 같은 특징을 가진다:

- NULL 값을 자동으로 제외함  
- 숫자 타입에서 최댓값을 효율적으로 계산  
- 별도의 필터링 없이 길이 컬럼에 바로 적용 가능

따라서 NULL 처리나 추가 조건이 필요 없으며, 단일 집계 쿼리로 해결 가능하다.

---

## 2. 쿼리 구조 분석

### (1) MAX(LENGTH)
```
MAX(LENGTH)
```
- LENGTH 컬럼에서 가장 큰 값을 반환  
- LENGTH가 NULL이어도 자동으로 무시됨  
- 문제 요구사항(가장 큰 물고기 길이)과 직접적으로 대응

---

### (2) CONCAT(MAX(LENGTH), 'cm')
```
CONCAT(MAX(LENGTH), 'cm')
```
- 숫자 값을 `"숫자 + cm"` 형태의 문자열로 변환  
- 출력 형식 요구사항을 만족하기 위한 처리

---

### (3) SELECT … AS MAX_LENGTH
```
SELECT CONCAT(MAX(LENGTH), 'cm') AS MAX_LENGTH
```
- 결과 컬럼명을 문제에서 요구하는 `MAX_LENGTH` 로 명시

---

## 3. 성능 관점

- `MAX()` 는 테이블 전체 스캔이 필요하지만, 연산 자체는 매우 가볍다.
- 물고기 길이 컬럼(LENGTH)에 인덱스가 있어도  
  일반적으로 집계함수(MAX)는 **인덱스만으로 처리 가능** → 성능적 이점이 있음.
- CONCAT은 단순 문자열 결합으로 비용이 거의 없음.
