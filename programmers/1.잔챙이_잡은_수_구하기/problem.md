# 잔챙이 잡은 수 구하기

## 1. 문제 설명

낚시 앱에서 사용하는 `FISH_INFO` 테이블에는 잡은 물고기들의 정보가 저장되어 있다.  
각 행은 한 마리 물고기에 해당하며, ID, 물고기 종류, 길이, 잡은 날짜 정보를 가진다.
단, **길이가 10cm 이하인 물고기**의 경우에는 실제 길이 대신 `LENGTH` 값이 `NULL` 로 저장된다.  
또한 `LENGTH` 컬럼이 전부 `NULL` 인 경우는 존재하지 않는다고 가정한다. 

이때, **잡은 물고기 중 길이가 10cm 이하인 물고기의 수**를 구하는 SQL을 작성하라.

---

## 2. 테이블 구조

`FISH_INFO` 테이블 스키마:

| 컬럼명     | 타입    | Nullable | 설명                     |
|-----------|---------|----------|--------------------------|
| `ID`      | INTEGER | FALSE    | 물고기 ID (기본 키)      |
| `FISH_TYPE` | INTEGER | FALSE  | 물고기 종류 (숫자 코드)  |
| `LENGTH`  | FLOAT   | TRUE     | 물고기 길이(cm), 잔챙이는 NULL |
| `TIME`    | DATE    | FALSE    | 물고기를 잡은 날짜       | 

**중요 조건**

- 길이 `10cm 초과` → `LENGTH` 에 실제 값 저장  
- 길이 `10cm 이하` → `LENGTH` 는 `NULL` 로 저장  
- `LENGTH` 컬럼 전체가 `NULL` 인 경우는 없음

---

## 3. 요구사항

다음 조건을 만족하는 SQL을 작성하라.

1. **길이가 10cm 이하인 물고기의 수**를 구한다.
   - 정의상 `LENGTH IS NULL` 인 행의 개수를 세면 된다. 
2. 결과 컬럼명은 반드시 `FISH_COUNT` 로 한다.
3. 출력 형식:

| FISH_COUNT |
|------------|
| (10cm 이하 물고기 수) |

---

## 4. 예시

예를 들어, `FISH_INFO` 테이블이 다음과 같다고 하자.  

| ID | FISH_TYPE | LENGTH | TIME       |
|----|-----------|--------|------------|
| 0  | 0         | 13.37  | 2021-12-04 |
| 1  | 0         | 50     | 2020-03-07 |
| 2  | 0         | 40     | 2020-03-07 |
| 3  | 1         | 43.33  | 2022-03-09 |
| 4  | 1         | NULL   | 2022-04-08 |
| 5  | 2         | NULL   | 2020-04-28 |

- ID 4, 5번 물고기의 `LENGTH` 가 `NULL` 이므로  
  → 두 마리는 길이 10cm 이하인 잔챙이

따라서 정답은:

| FISH_COUNT |
|------------|
| 2          |

---

## 5. 한 줄 요약

- **핵심 조건:** “10cm 이하”는 `LENGTH <= 10` 이 아니라 **`LENGTH IS NULL`** 로 표현된다.  
- 따라서, `LENGTH IS NULL` 인 행 개수를 `COUNT(*)` 또는 `COUNT(ID)` 로 세면 된다.
